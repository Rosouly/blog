本地运行

1. 安装 Hugo

首先，从 Hugo 官网 下载并安装适用于您操作系统的 Hugo。

2. 创建新的 Hugo 站点

打开命令行工具，然后输入以下命令创建一个新的 Hugo 站点：

hugo new site my-blog

这将在当前目录下创建一个名为 “my-blog” 的文件夹，其中包含 Hugo 站点的所有文件和目录。

3. 选择一个主题

浏览 Hugo 主题库，选择一个合适的主题。这里以本博客的主题为例： “Loveit” ，将其添加到你的站点中：

cd my-blog
git init
git submodule add https://github.com/dillonzq/LoveIt.git themes/example-theme

将主题添加到站点配置文件 config.toml 中：

theme = "example-theme"

4. 创建内容

使用以下命令创建一篇新的博客文章：

hugo new posts/my-first-post.md

这将在 content/posts 目录下创建一个名为 “my-first-post.md” 的 Markdown 文件。编辑该文件，添加您的博客内容。

5. 预览博客

在本地预览您的博客，输入以下命令：

hugo server

这将启动一个本地服务器，访问 http://localhost:1313 可以预览您的博客。

部署到 GitHub Pages

1. 在github创建仓库

在 GitHub 上创建一个新的仓库，例如：your-username.github.io（请替换为您的 GitHub 用户名）。

使用以下命令将站点的 public 目录添加到新仓库：

hugo
cd public
git init
git remote add origin https://github.com/your-username/your-username.github.io.git

2. 将站点推送到 GitHub：

git add .
git commit -m "Initial commit"
git branch -M main
git push -u origin main

3. 配置github page

  1. 登录到您的 GitHub 账户，然后转到您刚刚创建的 your-username.github.io 仓库。
  2. 点击仓库页面右上角的 “Settings”（设置）选项卡。
  3. 在设置页面中，向下滚动到 “GitHub Pages” 部分。
  4. 在 “Source”（来源）下拉菜单中，选择 “main” 分支。
  5. 点击 “Save”（保存）按钮。

4. 更新站点配置文件

打开您的 Hugo 站点目录中的 config.toml 文件，然后检查 baseURL 设置。将其更改为您的 GitHub Pages URL 

baseURL = "改为你实际的url"

5. 重新生成站点

在命令行中，导航到您的 Hugo 站点目录，然后运行 hugo 命令重新生成站点：

hugo

6. 推送更改到 GitHub

将更新后的静态文件推送到 GitHub：

cd public
git add .
git commit -m "Fix baseURL"
git push origin main

现在，您的博客应该已经部署到 GitHub Pages。访问 你网站的url 查看您的博客。

请注意，每次更新站点内容时，您都需要重新生成站点并将更改推送到 GitHub：

hugo
cd public
git add .
git commit -m "Update site"
git push origin main

这就是使用 Hugo 和 GitHub Pages 搭建博客的过程。祝您博客建设成功！
