任务一

成果：在浏览器分别访问 http://<k8s节点1的ip>/30000.com 和 http://<k8s节点2的ip>/30000.com 显示 [v1] Hello, Kubernetes!

前言：本次任务将涉及两个概念，一个是pod，一个是service。同时将会提供两个yaml文件，达到成果后，想办法请搞清楚这两个文件每一行的的含义，并理清整个过程。

检验方法：

  1. 试着用自己的话说出对pod和service的理解
  2. 继续任务二

先备条件/知识:

  1. 搭建了两个节点以上的k8s集群
  2. 构建golang镜像推送到dockerhub中，详情见：https://k8s-tutorials.pages.dev/container.html

pod

# hellok8s.yaml
apiVersion: v1
kind: Pod
metadata:
  name: hellok8s
  labels:
    app: hellok8s
spec:
  containers:
    - name: hellok8s-container
      image: rosylsq/hellok8s:v3 # 镜像请改成你自己的镜像

执行

kubectl apply -f hellok8s.yaml

service

# service.yaml
apiVersion: v1
kind: Service
metadata:
  name: service-hellok8s-nodeport
spec:
  type: NodePort 
  selector:
    app: hellok8s
  ports:
  - port: 3000
    nodePort: 30000
    targetPort: 3000

执行

kubectl apply -f service.yaml

任务二

成果：在浏览器分别访问 http://<k8s节点1的ip>/30000.com 和 http://<k8s节点2的ip>/30000.com 显示 [v1] Hello, Kubernetes!

前言：本次任务复用上次的service文件，将不再使用上次的pod的文件，多了一个deployment.yaml文件，涉及到概念是deployment

检验方法：

  1. 虽然两个任务都是达成同一个成果，比较两种实现方式有什么不同。
  2. 先kubectl get pod ，再删除前缀为hellok8s的pod，再执行kubectl get pod ，观察现象，搞清楚原因。
  3. 用自己的话说出deployment的作用。

deployment

先删除之前创建的pod：

# 参考命令
kubectl delete pod --all # 删除所有pod
kubectl get pod # 获取pod
kubectl delete pod <指定pod的名词> # 删除指定pod

# deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: hellok8s-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: hellok8s
  template:
    metadata:
      labels:
        app: hellok8s
    spec:
      containers:
        - image: rosylsq/hellok8s:v3
          name: hellok8s-container

执行

kubectl apply -f deployment.yaml

任务三

成果：在浏览器访问 http:///hello 显示 [v3] Hello, Kubernetes!，访问 http:/// 显示 nginx 默认页面。

前言：本次任务将涉及到一个新的概念，Ingress。Ingress 是 Kubernetes 提供的一种 HTTP/HTTPS 路由规则管理方法，可以用于在集群外部和内部服务之间提供路由。我们将通过 Ingress 为我们的应用创建一个访问入口。

检验方法：

  1. 试着用自己的话说出 Ingress 的作用和原理。
  2. 比较 Service 和 Ingress，他们的作用和运用场景有何不同。

下载ingress-controller

  1. 安装Helm
curl https://baltocdn.com/helm/signing.asc | sudo apt-key add -
sudo apt-get install apt-transport-https --yes
echo "deb https://baltocdn.com/helm/stable/debian/ all main" | sudo tee /etc/apt/sources.list.d/helm-stable-debian.list
sudo apt-get update
sudo apt-get install helm
  1. 添加 Helm 仓库
helm repo add ingress-nginx https://kubernetes.github.io/ingress-nginx
helm repo update
  1. 安装 NGINX Ingress 控制器：最后，你可以使用 Helm 安装 NGINX Ingress 控制器。在默认的 Kubernetes 命名空间（通常是 kube-system 或者 default）中安装它，或者在你选择的其他命名空间中安装。运行以下命令：
helm install ingress-nginx ingress-nginx/ingress-nginx

这将在你的集群中安装 NGINX Ingress 控制器。你可以通过以下命令检查安装情况：

kubectl get pods --all-namespaces -l app.kubernetes.io/name=ingress-nginx --watch
  1. 设定external ip 执行 kubectl get ingress 发现 Address 字段是空白的 执行：
kubectl edit service/ingress-nginx-controller

添加externalIPs字段, externalIPs为服务器内网ip

spec:
  allocateLoadBalancerNodePorts: true
  clusterIP: 10.109.248.219
  clusterIPs:
  - 10.109.248.219
  externalIPs:
  - 172.19.0.13
  externalTrafficPolicy: Cluster
  internalTrafficPolicy: Cluster

Ingress

首先，我们需要删除之前创建的所有 pod，deployment 和 service 资源。

kubectl delete deployment,service --all

然后，创建 hellok8s:v3 和 nginx 的 deployment 与 service 资源。需要注意的是，这次我们将 Service 的 type 设置为 ClusterIP。

hellok8s:v3 的端口映射为 3000:3000，nginx 的端口映射为 4000:80。这里的端口映射将在后续的 Ingress Route 规则中用到。

# hellok8s.yaml
---
apiVersion: v1
kind: Service
metadata:
  name: service-hellok8s-clusterip
spec:
  type: ClusterIP
  selector:
    app: hellok8s
  ports:
  - port: 3000
    targetPort: 3000
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: hellok8s-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: hellok8s
  template:
    metadata:
      labels:
        app: hellok8s
    spec:
      containers:
        - image: rosylsq/hellok8s:v3
          name: hellok8s-container

执行

kubectl apply -f hellok8s.yaml

# ingress.yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: hello-ingress
  annotations:
    # We are defining this annotation to prevent nginx
    # from redirecting requests to `https` for now
    nginx.ingress.kubernetes.io/ssl-redirect: "false"
    # 下面这行必不可少，否则ingress-controller无法识别，意思是指定使用ingress-nginx
    kubernetes.io/ingress.class: "nginx"
spec:
  rules:
    - http:
        paths:
          - path: /hello
            pathType: Prefix
            backend:
              service:
                name: service-hellok8s-clusterip
                port:
                  number: 3000

执行

kubectl apply -f ingress.yaml

访问 http://<公网ip>:80/hello (等价于http://<公网ip>/hello) 得到响应
